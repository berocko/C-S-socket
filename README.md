# 计算机网络实验：TCP/UDP Socket 并发通信

本目录包含一个简易的 Python Socket 实验项目，分别实现了：

- TCP 服务端与客户端通信
- UDP 服务端与客户端通信
- 两种并发处理模式：`多线程` 与 `线程池`

适合用于《计算机网络》课程中 C/S（Client/Server）模型、传输层协议与并发处理机制的实验演示。

## 目录结构

```text
code/
├── SimpleSocketS.py   # TCP 服务端
├── SimpleSocketC.py   # TCP 客户端
├── SimpleUDPS.py      # UDP 服务端
├── SimpleUDPC.py      # UDP 客户端
└── README.md
```

## 运行环境

- Python 3.8+
- 标准库即可（无需额外安装第三方依赖）

## 默认配置

四个脚本都使用相同地址：

- IP: `127.0.0.1`
- Port: `9999`

如需修改通信地址，请在对应文件中修改：

- `IP_PORT = ("127.0.0.1", 9999)`

## 一、TCP 实验

### 1. 启动 TCP 服务端

```bash
python3 SimpleSocketS.py
```

启动后可选择并发模式：

1. `多线程模式`：每个连接由一个线程处理
2. `线程池模式`：连接任务提交到线程池处理

### 2. 启动 TCP 客户端

另开一个终端执行：

```bash
python3 SimpleSocketC.py
```

客户端输入消息后，服务端会回显确认信息。
输入 `exit` 可退出客户端。

## 二、UDP 实验

### 1. 启动 UDP 服务端

```bash
python3 SimpleUDPS.py
```

启动后可选择并发模式：

1. `多线程模式`：每个数据包创建一个线程处理
2. `线程池模式`：数据包处理任务提交到线程池

### 2. 启动 UDP 客户端

另开一个终端执行：

```bash
python3 SimpleUDPC.py
```

客户端输入消息后，服务端返回 `UDP服务器已收到: xxx`。
输入 `exit` 可退出客户端。

## 并发模式说明

- `thread`（多线程）
  - 优点：实现直观
  - 缺点：高并发下线程创建/销毁开销大

- `pool`（线程池）
  - 优点：线程可复用，资源利用更稳定
  - 缺点：需要合理设置线程池大小

默认线程池大小为 `10`（`DEFAULT_WORKERS = 10`）。

## 交互与输出说明

- 服务端会打印客户端地址与收到的消息。
- 客户端会在后台线程接收服务端响应，并在终端输出。
- 按 `Ctrl + C` 可中断服务端运行。

## 常见问题

1. `Address already in use`

说明端口被占用。可先结束旧进程，或修改 `IP_PORT` 端口。

2. 客户端无法连接

请确认：

- 服务端已先启动
- 客户端与服务端 IP/端口一致
- 本机防火墙未拦截本地回环通信

## 实验建议

- 分别在 `thread` 与 `pool` 模式下运行多个客户端，观察吞吐与响应差异。
- 对比 TCP 与 UDP 在连接管理、可靠性、消息处理方式上的不同。
- 结合抓包工具（如 Wireshark）进一步分析传输层行为。
